|Identificador|HU_001_AUTENTICACION_ESTUDIANTE| 
|--------------|-------------------------------|
|Título|Autenticación estudiante|
|Descripción|Como estudiante, quiero poder registrarme e iniciar sesión en la plataforma, para acceder a las funciones de inscripción y consulta de cursos.|
|Criterios de aceptación|#El sistema debe permitir el registro ingresando los campos: nombre, apellidos, número de estudiante, correo institucional y contraseña.|
||#El sistema debe validar que el correo pertenezca al dominio institucional antes de registrar al usuario.|
||#El sistema debe validar que todos los campos obligatorios estén completos y cumplan el formato establecido (por ejemplo: correo válido, longitud mínima de contraseña).|
||#El sistema debe impedir el registro si el número de estudiante ya se encuentra registrado.|
||#El sistema debe impedir el registro si el correo institucional ya se encuentra registrado.|
||#El sistema debe permitir inicio de sesión con correo institucional y contraseña válidos previamente registrados.|
||#Si las credenciales son incorrectas, debe mostrarse el mensaje: “Usuario o contraseña incorrectos”.|
||#Si el usuario intenta iniciar sesión con un correo no registrado, el sistema debe mostrar un mensaje indicando que el usuario no existe.|
||#Tras autenticación exitosa, el sistema debe redirigir al panel principal del estudiante.|
||#El sistema debe almacenar las contraseñas de forma cifrada en la base de datos.|
||#El sistema debe mantener la sesión activa mientras el usuario interactúe con la plataforma y hasta un tiempo de expiración configurado sin actividad.|
||#El sistema debe permitir al estudiante cerrar sesión de forma explícita desde la interfaz.|

|Identificador|HU_004_INSCRIPCION_CURSO_REGULAR| 
|--------------|--------------------------------|
|Título|Inscripción en curso regular|
|Descripción|Como estudiante, quiero inscribirme en un curso regular seleccionando un horario disponible, para poder cursarlo en el periodo académico.|
|Criterios de aceptación|#El sistema debe requerir que el estudiante tenga sesión activa para realizar la inscripción.|
||#El sistema debe permitir al estudiante seleccionar un curso regular y uno de sus horarios disponibles para completar la inscripción.|
||#Antes de confirmar la inscripción, el sistema debe validar que el horario seleccionado tenga al menos 1 cupo disponible.|
||#Si no hay cupos disponibles en el horario seleccionado, el sistema debe impedir la inscripción y mostrar un mensaje indicando que el cupo está agotado.|
||#Si la validación de cupos es exitosa, el sistema debe registrar la inscripción, asociando estudiante, curso y horario.|
||#Al registrar la inscripción, el sistema debe actualizar el número de cupos disponibles del horario, disminuyéndolo en 1.|
||#El sistema debe impedir que un estudiante se inscriba más de una vez al mismo curso y horario.|
||#El sistema debe notificar al estudiante en la interfaz cuando la inscripción se haya completado exitosamente.|
||#El sistema debe evitar sobrepasar el cupo máximo del horario incluso ante múltiples solicitudes concurrentes de inscripción.|
||#En caso de error durante el registro de la inscripción, el sistema no debe modificar los cupos disponibles y debe mostrar un mensaje de error al estudiante.|

|Identificador|HU_005_INSCRIPCION_PAGO_CURSO_EXTENSION| 
|--------------|----------------------------------------|
|Título|Inscripción y pago de curso de extensión|
|Descripción|Como estudiante, quiero inscribirme en un curso de extensión y realizar el pago en línea, para asegurar mi cupo en dicho curso.|
|Criterios de aceptación|#El sistema debe requerir que el estudiante tenga sesión activa para inscribirse en un curso de extensión.|
||#El sistema debe permitir que el estudiante seleccione un curso identificado como “curso de extensión” y uno de sus horarios disponibles.|
||#Antes de iniciar el proceso de pago, el sistema debe validar que el horario seleccionado tenga al menos 1 cupo disponible.|
||#El sistema debe mostrar el valor a pagar del curso de extensión antes de redirigir o invocar el servicio de pagos en línea.|
||#El sistema debe integrarse con el servicio de pagos en línea para procesar la transacción asociada al curso de extensión.|
||#El sistema debe registrar el resultado de la transacción de pago (aprobado o rechazado) retornado por el servicio de pagos.|
||#Si el pago es aprobado, el sistema debe registrar la inscripción del estudiante al curso de extensión, asociando estudiante, curso, horario y referencia de pago.|
||#Si el pago es aprobado, el sistema debe actualizar los cupos disponibles del horario, disminuyéndolos en 1.|
||#Si el pago es rechazado o no se completa, el sistema no debe registrar la inscripción ni modificar los cupos disponibles.|
||#El sistema debe mostrar un mensaje claro al estudiante cuando el pago sea aprobado y la inscripción confirmada.|
||#El sistema debe mostrar un mensaje claro al estudiante cuando el pago sea rechazado o falle, indicando que la inscripción no se ha realizado.|
||#El sistema debe evitar sobrepasar el cupo máximo del horario, incluso ante múltiples intentos concurrentes de pago para el mismo curso y horario.|
